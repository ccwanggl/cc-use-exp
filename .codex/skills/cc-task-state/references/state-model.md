# State Model

推荐状态：

- `需求中`：只有需求或想法，尚未达到可开工程度
- `待开始`：范围和目标已基本明确，但尚未开始实施
- `进行中`：当前主线正在推进
- `被打断`：原任务被其他事项临时插入，需记录 checkpoint
- `阻塞中`：因依赖、环境、决策或外部输入暂停
- `待恢复`：阻塞或插入原因已解除，等待继续
- `已完成`：任务完成，应移入 `archived/`
- `已取消`：不再继续，应移入 `archived/`

推荐迁移：

- `需求中 -> 待开始 -> 进行中 -> 已完成`
- `进行中 -> 被打断 -> 待恢复 -> 进行中`
- `进行中/待开始 -> 阻塞中 -> 待恢复`
- 任何非终态都可进入 `已取消`

补充约束：

- 同一项目同一时刻最多一个主任务处于 `进行中`
- 短期 fix 默认不新建独立任务，先记入主任务的“中断记录”
- 用户明确说“还没开始”“先记一下”“等主线收尾再做”时，不得标记为 `进行中`
